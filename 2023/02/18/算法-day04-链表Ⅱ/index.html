leetcode24：两两交换链表中的节点

给你一个链表，两两交换其中相邻的节点，并返回交换后链表的头节点。你必须在不修改节点内部的值的情况下完成本题（即，只能进行节点交换）。

这是一个中等的链表交换题目，在数组中交换元素需要一个临时变量存储，然后交换两个值，在链表中因为是有着next指针的，通过对next指针的指向进行操作来达到交换的目的。对于链表的增删最好引入虚拟头节点来进行操作的统一性。

leetcode19:删除链表的倒数第N个节点

给你一个链表，删除链表的倒数第 n 个结点，并且返回链表的头结点。

这个题目是中等难度的问题，暴力解法并不难，直接while遍历一遍得到总的节点数，然后第二次遍历到size-n处 释放掉next节点并且连接next->next即可。但是题目进阶要求一次遍历，想到了使用快慢指针，让快指针先走N步，然后快慢指针同时走，一直走到快指针为nullptr此时慢指针的后面就是要删除的哪个节点。

面试题 02.07. 链表相交

给你两个单链表的头节点 headA 和 headB ，请你找出并返回两个单链表相交的起始节点。如果两个链表没有交点，返回 null 。

这是道简单题型，直接暴力的话就是可以利用hashmap做，只需要找到第一个相同的节点就停止。

这个时间复杂度为O(n)，空间复杂度O(n)。

如果要求空间复杂度为O(1)的情况下，可以先遍历获取长度，然后右对齐进行比较（等长开始比较，多出的一定不是相同的节点，一定是右对齐），当比较到节点地址相同就是要找的节点了。

leetcode142： 环形链表 II

给定一个链表的头节点 head ，返回链表开始入环的第一个节点。 如果链表无环，则返回 null。

如果链表中有某个节点，可以通过连续跟踪 next 指针再次到达，则链表中存在环。 为了表示给定链表中的环，评测系统内部使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。如果 pos 是 -1，则在该链表中没有环。注意：pos 不作为参数进行传递，仅仅是为了标识链表的实际情况。

不允许修改 链表。

这个题目应该是接触的第一个需要数学推导的题目，这个题目要借助快慢指针，快指针走两步，慢指针走一步，为什么这么走呢，因为一旦进入环里，两个节点将以一步的差距慢慢的靠近直到重合，假设从头结点到环形入口节点的节点数为x。 环形入口节点到 fast指针与slow指针相遇节点 节点数为y。 从相遇节点 再到环形入口节点节点数为 z。

slow走了x+y，fast走了x+y+n(y+z),n就是快指针在环内转的圈数，

快指针走的是慢指针的二倍，所以 2x+2y = x+y+n(y+z) 即 x+y = n(y+z) x = n(y+z)-y 所以相遇后 ，将指针一个退回原点，正常走相遇点就是交点。
